## ✨ Key Features

### 1️⃣ Personalized Premiums (AI/ML)

```
Same city, different drivers, DIFFERENT premiums:

Mumbai:
  Rajesh (₹1,200/day, 50h/week)  → ₹43.14/week (0.75% of income)
  Ankita (₹1,000/day, 35h/week)  → ₹25.01/week (0.52% of income)
  Neha   (₹650/day, 18h/week)    → ₹12.19/week (0.39% of income)

Why different?
  ✓ Different earnings (primary factor)
  ✓ Different activity levels (Worker Activity Index)
  ✓ Progressive pricing structure (high-risk gets lower rates)
```

### 2️⃣ 5 Automated Triggers (Real APIs)

```
Monitor for disruptions:
  1. IMD Weather Alerts    → 60-80% income loss
  2. AQI Pollution Spikes  → 30-70% income loss
  3. Platform Outages      → 40-60% income loss
  4. Fuel Price Spikes     → 35% income loss
  5. Civil Disruptions     → 70-90% income loss

All running in PARALLEL every 30 seconds.
Zero human intervention.
```

### 3️⃣ Zero-Touch Claims

```
Tuesday morning: Heavy rain detected

System automatically:
  1. Verifies disruption via IMD API
  2. Identifies affected drivers in zone
  3. Pulls day-end earnings from platform
  4. Calculates: 70% × income_loss × compliance_factor
  5. Checks: 2 anti-fraud gates
  6. Initiates payout via UPI

By 7 AM Wednesday: ₹₹₹ in driver's account

No claim form. No assessor. No waiting.
```

### 4️⃣ Two-Gate Anti-Fraud Formula

```
Gate 1: Hours Worked (Compliance Factor)
  Required: 50% of normal working hours
  Actual: 5 hours
  Score: 1.0 (exceeds requirement)
  
Gate 2: Minimum Orders
  Required: 1-2 orders (depends on severity)
  Actual: 8 orders completed
  Score: ✓ Pass

Final Payout = 0.70 × income_loss × 1.0 = ₹490
                ↑
           Co-participation incentivizes
           driver to attempt work
```
